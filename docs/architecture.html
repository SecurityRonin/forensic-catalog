forensicnomicon — Data Schema

The 5 compile-time typed layers and how they relate — all data in &'static, zero heap allocation, no_std compatible

Data schema — 5 typed layers

ArtifactDescriptor — the hub

  • • 6,554 entries: 361 hand-curated + 6,193 generated
  • • Generated from 7 corpora: KAPE (2,422), ForensicArtifacts (2,545), EVTX (995), Velociraptor (122), RECmd (44), browsers (37), NirSoft (22)
  • • 12-field struct: id, name, meaning, triage_priority, os_scope, key_path, file_path, mitre_techniques, sources, related, decoder, carving_sig
  • • All data in &'static — zero heap allocation, no_std compatible

InvestigationPath + Step

  • • INVESTIGATION_PATHS (6) — ATT&CK-tactic chains: lateral_movement, credential_harvesting, persistence, data_exfiltration, execution_trace, defense_evasion
  • • PLAYBOOKS (5) — scenario checklists: ransomware, data_breach, bec, insider, supply_chain
  • • InvestigationStep.artifact_id is a &'static str FK into CATALOG
  • • unlocks: &[&str] — branching: one step unlocks N follow-on artifacts

LolbasEntry + AbusableSite

  • • LolbasEntry: 6 parallel catalogs (Windows, Linux, macOS, Cmdlets, MMC, WMI) — 187/478/141/187+/63/30 entries
  • • use_cases: u16 bitmask — UC_EXECUTE | UC_DOWNLOAD | UC_PROXY | UC_BYPASS | UC_CREDENTIALS | UC_RECON | UC_PERSIST | UC_NETWORK
  • • AbusableSite: 54 sites · SiteCategory + BlockingRisk enums
  • • Parallel catalogs — no compile-time FK to ArtifactDescriptor; share MITRE technique strings

ArtifactProfile — enrichment

  • • id: &'static str FK into CATALOG — single source of truth for both evidence + volatility
  • • EvidenceStrength: Definitive | Strong | Moderate | Circumstantial | Unreliable
  • • VolatilityClass: LiveMemory | KernelOS | UserProcess | SystemConfig | Residual
  • • evidence_for(id) · volatility_for(id) · acquisition_order() (RFC 3227)
